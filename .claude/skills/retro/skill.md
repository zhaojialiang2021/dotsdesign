# /retro

分析本周工作，统计设计系统推进情况，输出改进建议。

## 执行流程

### 1. 获取设计系统快照

运行 `/audit`，获取当前组件和页面状态分布。

### 2. 分析本周改动

从 git log 提取本周（最近 7 天）的提交记录，统计：

- 组件状态推进数：placeholder → draft → complete
- 页面文档新增/更新数
- 令牌违规修复数
- 代码改动文件数
- 新增/删除代码行数

### 3. 识别模式

- 本周集中在哪类工作上？（组件定义 / 页面绘制 / 代码实现 / 修复）
- 是否有阻塞项反复出现？
- 令牌违规是否在减少？

### 4. 输出周回顾

```
## 周回顾 ({date_range})

### 数据
- 提交数：{N}
- 组件推进：{placeholder→draft→complete 数}
- 页面进展：{新增/更新 数}
- 令牌修复：{N} 个

### 模式
[本周工作重心和趋势]

### 做得好的
[具体事项]

### 可以改进的
[具体事项 + 建议]

### 下周建议
- [优先级排序的推进建议]
```

### 5. 更新上下文

如果 `.claude/context.md` 存在，更新其中的"设计系统进度"和"待推进"部分。
