# /inspect {name}

对照 Figma 设计稿和设计令牌，对已实现的页面做视觉质检。

## 执行流程

### 1. 读取文档（强制）

1. `references/design-tokens.md`
2. `references/design-principles.md`
3. `references/haptics.md`
4. 目标页面文档或组件文档

### 2. 获取参照基准

两个参照源，优先 Paper 画板：

- **Paper 画板**（优先）：如果页面对应有 Paper 画板，截图作为基准
- **Figma**（兜底）：Paper 画板不存在时从 Figma 获取设计稿截图。注意：Figma 是旧版参考，不作为真相源

### 3. 获取实现截图

- 启动 dev server（如未运行）：`pnpm dev`
- 截取实现页面的当前状态

### 4. 10 类检查

逐类检查，每项标注通过/不通过/不适用。

---

#### 1. 令牌一致性
- [ ] 所有颜色使用 `var(--token-name)` 而非硬编码 hex
- [ ] 间距使用 `var(--space-N)` 而非魔法数字
- [ ] 圆角使用 `var(--radius-N)` 而非硬编码
- [ ] 阴影使用 `var(--shadow-modal)` 而非自定义值

#### 2. 字号合规
- [ ] 仅使用 PingFang SC
- [ ] 字号来自令牌表 10 级，无中间值
- [ ] 字重仅 400 和 500
- [ ] 数字场景使用 tabular-nums

#### 3. 间距合规
- [ ] 所有间距来自 Space-1 到 Space-10
- [ ] 无非梯度值（7px、18px 等）

#### 4. 圆角合规
- [ ] 仅 6/8/12/20/9999px 五种
- [ ] 用途匹配：图片 6px、导航项 8px、内容卡 12px、气泡/容器卡 20px、Pill 9999px

#### 5. 颜色语义
- [ ] 品牌色只做点缀
- [ ] 卡片只用 BG-0
- [ ] 装饰色只做图标底色
- [ ] 分割线默认 NonOpaque
- [ ] AI 回应气泡使用 Brand-Blue-Light 背景 + Brand-Blue-Border 边框

#### 6. 交互状态
- [ ] 三态完整：Default / Active / Disabled
- [ ] Active 态规则正确（背景降一级，Label 透明度 0.6）
- [ ] 触觉意图已在文档中定义
- [ ] 过渡时间使用令牌（Duration-Fast/Normal/Slow）

#### 7. AI 内容区分
- [ ] AI 回应与用户输入视觉上可区分
- [ ] AI 内容标注"由 AI 生成"
- [ ] AI 回应 loading 状态已实现
- [ ] 无欺骗性 AI 模式

#### 8. 多宽度适配
- [ ] 内容最大宽度 680px
- [ ] Tablet 侧边栏 ~345px
- [ ] 响应式断点工作正常

#### 9. 无障碍
- [ ] 触控目标最小 44px
- [ ] 文字对比度 >= 4.5:1
- [ ] 语义化 HTML 标签
- [ ] 图片有 alt 文本

#### 10. 动效合规
- [ ] 时长来自令牌（150/250/350ms）
- [ ] 曲线来自令牌（Default/Spring/EaseOut）
- [ ] 无装饰性动画

#### 11. 触觉反馈
- [ ] 每个可交互元素触觉意图已定义
- [ ] 触觉意图名来自令牌表（tick/light/medium/warning/success）
- [ ] 组件文档中的触觉声明与 haptics.md 中的映射一致
- [ ] 降级策略已定义（无振动马达设备的视觉反馈替代）

---

### 5. 输出质检报告

```
## {name} 视觉质检

| 类别 | 通过 | 不通过 | 不适用 |
|------|------|--------|--------|
| 令牌一致性 | 3 | 1 | 0 |
| ... | | | |

### 不通过项
[每项：偏差描述 + 预期值(令牌) + 实际值 + 修复建议]
```

### 6. 可选自动修复

对可自动修复的项（令牌替换、间距调整等），询问用户是否自动修复。修复后重新检查。

### 7. 建议下一步

- 如果有不通过项：修复后重新 `/inspect`
- 如果全部通过：运行 `/ship` 发货
