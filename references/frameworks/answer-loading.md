# Answer Loading 回答加载框架

Answer Loading 是点点对话页里“回答不是瞬间出现，而是有可感知过程”的通用框架。它覆盖发送后判断、content 回复、连续长思考、工具调用、快速回答、response、思考过程半层和来源半层。

该框架只定义可复用逻辑和视觉规则；具体 demo 的开关、汇报控制器和讲解文案放在页面文档里。

## 核心原则

- 对话页能力必须基于真实 `DottedDemoScreen` 页面模板，复用状态栏、导航栏、消息流和底部输入区。
- 回答加载是消息流的一部分，不是独立说明卡。思考、工具调用和 response 都必须在真实消息位置上演进。
- 同一条思考链路里的阶段切换必须连贯：旧内容先收起或结束，再出现新阶段；不能把长文本和短文本叠在一起过渡。
- 高度和背景由内容自然撑开。禁止按字符数估算高度、禁止用固定空白兜底，避免文本和背景错位。
- 输出时只允许当前内容动，已经完成的内容保持稳定。完成态的标题、状态 icon 和连接线不能跳闪。

## 页面基座

点点对话 demo 使用 393×852px 手机画板，外壳圆角 48px，外部 8px 纯白外描边，浅色背景使用 `Bg`。顶部是 iOS 状态栏和点点导航；底部是输入区渐隐底座、48px 输入框、`内容由AI生成` 和 home indicator。

输入区背后使用独立透明渐变遮罩层：`#F5F5F5` 三段 stop 为 `0% / 0%`、`18% / 80%`、`36% / 100%`，36% 后保持 100%。渐变层不做背景模糊；输入框自身使用 `#FCFCFC` 纯色背景和 25px backdrop blur。输入框父级不能使用会隔离采样的合成上下文。

## 消息与气泡

用户气泡右对齐，背景为弱灰填充，文字 16px / 28px。单行圆角 18px，多行圆角 22px。用户气泡内每一行文字下方都有 0.75px 虚线，dash 2px / gap 2px，颜色使用 `Separator 2`，虚线跟随真实文本宽度，不写死两条线。

Dots 普通气泡左对齐，白底，文字 16px / 28px。response 卡片使用白底、0.5px 分隔色描边、22px 圆角；response 不做明显位移，只做轻淡入，并在正常文档流里向下拓展。

流式输出保留 B 版：文本按字符进入文档流，最后约 10 个字符做透明度梯度，末尾约 34%，越往前越接近 100%。输出完成后用 760ms 把尾部透明度慢慢恢复到 100%，不能直接跳变。

## 长思考链路

默认长思考链路顺序为：

1. 判断
2. think
3. content
4. think
5. tool call
6. think
7. tool call
8. think
9. response
10. 完成

`content` 是发送后第一条 Dots 回复气泡。content 使用快速字符流式输出和轻量浮现；结束后停留 600ms 再进入后续思考。

连续堆叠方案是当前主方案：已经出现过的思考过程保留在同一脑雾模块里，新过程追加在下方；上一过程进入收起态，只保留标题行和状态 icon。上一项必须先收起详情，再把 Lottie/emoji 切换为完成对勾。切换过程用原地溶解，不做明显缩放或位移。

连续堆叠脑雾顶部固定展示「正在思考」入口。入口不参与条目滚动，入口与下方内容间距 8px。只要当前标题或正文还没输出完，入口文字做单条慢速扫光：光带从左到右扫完后停顿 2s 再开始下一轮。箭头直接跟在文字后，间距 4px，样式与完成态箭头一致。

条目滚动层最大高度为 156px。超过后只在思考区域内部滚动；正在输出的当前内容必须完整展示。滚动时使用线性贴底补间，不能一段一段跳。只有发生内部滚动后才显示顶部渐变蒙层，未滚动时不显示蒙层。

连接线使用 1px 竖向虚线，颜色 `Separator 2`，dash/gap 1.5px。连接线只连接完成收起行之间，避开图形本体，距离上下圆形各保留 2px。当前展开正文下方、最后一个节点、response 正文左侧都不显示连接线。

## 状态 Icon

当前最新一项播放对应 Lottie：think 使用 `cloud.json`，tool call 可用 `pen.json` 或 `glass.json`，搜索类 tool call 使用 `glass.json`。非当前行停在 JSON 第一帧。

完成态使用设计稿导出的完成对勾资源：32×32px，浅灰圆形底、弱描边圆环和弱勾线。不要用 CSS 手画大对勾替代。Lottie 到对勾的过渡时长 640-680ms，先淡出 Lottie，再淡入对勾。

## Tool Call 来源展示

Tool call 有两种来源展示样式：

- 胶囊样式：外部只展示轻量来源 pill，适合对话流降噪。
- 信息卡样式：外部展示两个信息卡，左侧为小红书，右侧为全网。

信息卡默认数据：

- 小红书：`14 篇笔记和评论`
- 全网：`8 个网页`

数字从 0 滚动到目标值，节奏需要有真实加载感，时快时慢。来源项逐个出现；第一个胶囊或卡片完成后，间隔 500ms 出第二个；最后一个结束后不额外停顿，直接进入下一思考。

信息卡内文字左对齐，图片右对齐，中间间距 auto，整体在卡片高度内垂直居中。卡片背景不额外加色块，只保留设计稿边框和必要弱阴影。右侧图标动画只控制“动不动”，不控制是否展示；默认展示但不动。

右侧图标动效使用横向顶替：初始位置保持设计稿叠放；前一个图标渐隐消失，后一个图标横向顶到前一个位置，新的后置图标在原后置位置渐显。动画期间始终保持两个图标的叠放关系，不能出现只剩一个图标的空档。

## 快速回答链路

快速回答是独立链路，不复用长思考当前帧。content 输出完成后，输入框上方出现「获得快速回答」按钮；按钮从下方渐隐浮现。点击后按钮立即消失，直接进入快速思考，不再播放判断 cloud。

快速思考只展示单条思考标题，文案分阶段渐隐渐现替换，例如「搜索伊犁经典玩法」到「搜索核心景点信息」。文字颜色不做尾部渐变。最后一条思考完成后停留 1s，再进入快速 response。快速 response 也使用 B 版流式输出。

快速回答完成态文案为：

`思考完成，参考小红书与全网12人真实经验`

快速回答来源区只展示参考来源，不展示思考过程和上方框架：

- `参考来源 · 12`
- `笔记 6`
- `评论 4`
- `全网 2`

## 完成长思考 Response

长思考完成态状态行为：

`思考完成，参考小红书与全网22人真实经验`

长思考来源区：

- `参考来源 · 22`
- `笔记 10`
- `评论 4`
- `全网 8`

response 状态行和 response 正文必须共用同一个卡片壳。状态行从脑雾过渡到白色 response 背景时只做清晰度和背景透明度过渡，不做布局切换。正文开始输出时，状态行位置不能上下跳。

response 正文可包含图片，但有图内容里的引用符号不展示；去掉引用符后不能留下额外空格。

## 思考过程半层

思考中点击脑雾打开「思考中」半层；思考结束后标题更新为「思考过程」或「思考完成」。半层从底部拉起，低半层 639px，高半层 798px。半层上滑进入高半层，下拉超过 96px 关闭。

思考半层使用白底过程卡片、32×32 状态列和虚线连接。所有过程默认收起，点击有内容的类目展开；多个类目可以同时展开，不互斥。没有内容的类目不显示箭头，也不支持点击。

展开/收起使用高度和透明度的“呼吸展开”，时长 400ms，曲线 `cubic-bezier(0.22, 0.61, 0.36, 1)`，不做上下位移。

## 来源半层

response 状态行点击打开来源半层。来源半层复用 Sheet 拉起、遮罩、grabber 和高半层逻辑。

来源导航在滚动到参考来源区域后吸顶：标题和筛选胶囊组成吸顶组，背景与半层一致，防止下方内容穿透。

## 真实感 Loading Demo

真实感 loading 是从长思考拆出的独立 demo，用来验证单个 tool call 的静态页面和动画。

页面仍沿用点点回答 loading 的手机壳和汇报控制区域。对话内容替换为电视机场景：

- 用户气泡：`6千以内电视机推荐`
- Dots 气泡：`先看看去除营销软广后，小红书内真人推荐最多的电视机型号；然后帮你研究怎么选。`
- 当前 loading 标题：`搜索 6千以内电视机推荐`
- 数字：`已查找 1165 条真人经验`、`已过滤 223 条营销软广`
- 下一条收起标题：`研究电视机怎么选`

数字从 0 滚动到目标值，时长 4s，节奏时快时慢。数字滚动完成后，搜索状态 icon 在原位置替换为完成对勾；完成对勾样式必须与上一个已完成思考项一致。

下方卡片动画使用叠卡轮转：最大卡位移到中间卡片位置，中间卡片位移到第三张位置，第三张沿路径淡出，第一个位置新长出一张。icon 与卡片绑定，跟随卡片移动，不能在卡片切换中跳位。动画结束后数据展示停留 1s，再收起成第一条完成态样式；标题颜色也切到已完成标题颜色。
