# Framework-A: 聊天流

> status: draft
> last_updated: 2026-07-08

## 描述

Dots 核心交互框架。用户输入 + AI 回应的聊天式对话流，覆盖所有“问 AI 拿答案”的场景。当前页面基座以 Figma `think 方案` node `3:4417` 为准。

## 结构

```
┌────────────────────────┐
│ iOS StatusBar          │
├────────────────────────┤
│ Back  Avatar 点点  More │ ← 头像导航
├────────────────────────┤
│                        │
│  Dots 白底问候气泡       │
│                        │
│        用户灰底气泡      │
│                        │
│  判断动画 / Dots 回复    │
│                        │
├────────────────────────┤
│  玻璃渐隐底座             │
│  语音 + 输入占位 + 工具    │

└────────────────────────┘
```

## 关键规则

- 页面级 demo 和 pattern demo 必须复用真实页面模板 `DottedDemoScreen`，不能重建假手机壳、假输入栏或假气泡。
- 顶部导航使用头像 + 标题「点点」，而不是只有居中标题的旧导航。
- 用户气泡右对齐，背景 `rgba(0,0,0,0.08)`，圆角 18px，内部两条 dash 2px / gap 2px 的弱虚线对齐文本行。
- Dots 气泡左对齐，白底，圆角 18px，无描边、无引脚。
- 消息列表自动滚动到最新内容。
- AI 回应 thinking 态：在 Dots 回复位置展示 cloud 判断 Lottie，不显示文字，不使用普通 loading 圆点。
- AI 回应 judging 态：cloud 脑雾横向展开，右侧简单判断文案按字符流式输出；文字输出期间不扫光。
- AI 回应 judgingHold 态：简单判断文案完整后触发慢速扫光，保持 2s。
- AI 回应 streaming 态：`judgingHold` 结束后自动接出 Dots 回复气泡，气泡从下方 32px 柔和位移浮现，同时按字符连续打出正式回答；位移动效只能有一条 transform/opacity 动画，形成完整 typewriter 动作，不显示硬光标或尾点。
- AI 回应 think 态：正式回复完成后，可在下一条 Dots 消息位置追加思考脑雾；标题和正文都按字符流式输出，只有标题在完整输出后触发慢速扫光。脑雾背景透明度为 90%，随标题进度横向即时展开，纵向高度由正文、检索结果等真实 DOM 内容自然撑开，不能按正文行数估算固定高度。
- AI 回应 toolcall 态：think 完成后同一条脑雾渐变切换到 toolcall，动画切换为 `pen.json`，标题和正文重新按字符流式输出，只有标题扫光。
- AI 回应紧凑 think 态：toolcall 完成后同一条脑雾渐变切回 think，动画切换为 `cloud.json`，只流式输出标题，不展示正文。
- AI 回应第二段 toolcall 态：紧凑 think 完成后同一条脑雾渐变切换到 toolcall，动画切换为 `glass.json`，标题和检索结果重新按字符流式输出，只有标题扫光。
- AI 回应第三段 think 态：检索 toolcall 完成后同一条脑雾渐变切回 think，动画切换为 `cloud.json`，标题和正文重新按字符流式输出，只有标题扫光。
- AI 回应 think/toolcall 连贯性：阶段之间保持在同一条脑雾里，上一个阶段正文输出完成后停顿 1s，再切换到下一个阶段，只保留同一脑雾容器自身的渐变。
- AI 回应 response 态：第三段 think 完成后停顿 1s，思考脑雾消失，再追加最终 response 卡片；卡片沿用气泡 32px / 1200ms 的柔和浮现，正文继续按字符流式输出，并在消息流内向下拓展。
- AI 回应 complete 态：保留普通 Dots 气泡和完整 response 卡片；底部复制/分享/不喜欢/刷新操作栏只在 response 正文输出结束后展示，但页面不自动跳到回答结尾。
- 思考过程中，思考脑雾整体可点击打开 `8:1257`「思考中」半层；半层按当前已经完成/正在输出的 think/toolcall 顺序展示过程，当前最新行用正在播放的 Lottie 作为 24×24 状态位，内容在半层内按正常文档流向下拓展；若半层打开期间思考结束，标题更新为「思考结束」，`搜索上海酒店真实入住体验` 切换为 Figma `116:6060` 的结果卡样式，最后一个 think 行继续保留完整正文。
- response 状态行打开思考/来源半层；半层内容上滑时进入高半层，向下拖动超过 96px 时直接关闭半层；类目默认全部收起并支持多项同时展开，过程数据必须映射前面的 think/toolcall/response 链路，不能独立编排。
- 底部输入固定在玻璃渐隐底座内，输入框高 48px，保留语音、相机、添加三个入口；对话流延伸到页面底部，通过底部留白和渐隐遮罩处理内容与输入框的覆盖关系。
- response 长内容超出可视区域且用户不在回答底部时，在输入区上方显示 36×36 “至底”悬浮按钮；点击后滚动到回答底部并隐藏。
- AI 回应失败态暂未进入当前 demo；后续需要时先补状态规格。

## 多宽度适配

| 断点 | 变化 | 说明 |
|------|------|------|
| Mobile 393px | 无变化 | 标准体验 |
| Tablet | 保持手机画板预览，不拉伸页面 | 文档站外层可居中展示 |
| Desktop | 保持手机画板预览，不拉伸页面 | 说明面板放在手机画板外侧 |
