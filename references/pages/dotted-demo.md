# 点点demo

## 需求背景

点点demo 是 Dots 独立设计系统的页面样本，用来验证「Figma 规范 → 规格文档 → 页面 demo → GitHub → Vercel 预览」这条流程能否跑通。

当前视觉真相源切换为 Figma `think 方案` 节点 `3:4417`。旧版 `Dots 规范` 中的快捷提示、系统键盘、Skill 选中态和护肤 AI 卡片不再作为当前对话页框架基座；如果后续要恢复这些能力，必须先重新进入组件规格流程，而不是在页面 demo 私自拼回旧结构。

## 页面结构

1. iOS 状态栏：时间 9:41，右侧信号、Wi-Fi、电池。
2. 顶部导航：左侧返回，头像 + 标题「点点」，右侧更多。
3. 对话区：真实消息流容器，复用 `DotsMessageBubble`；项目 demo 不展示默认问候气泡。
4. 底部输入区：玻璃渐隐底座 + 48px 输入框。
5. 输入框内容：左侧语音图标，占位文案「发消息或按住说话...」，右侧相机和添加。
6. 底部标注：`内容由AI生成` + iOS home indicator。

## 视觉规格

- 画板为 393×852px，外壳圆角 48px，使用 iOS 连续圆角 / squircle 裁切，外部 8px 纯白色外描边，不占用内部画布尺寸，背景为 `Bg0` / `#F5F5F5`。
- 顶部基座高 110px：系统状态栏 54px，导航栏 56px，背景与页面背景一致。
- 导航栏左右 padding 16px；返回、更多图标均为 24×24px；头像为 24×24px，圆角 8px；标题 16px / 24px / Medium。
- 对话区从 y=110px 开始一直延伸到页面底部，左右 padding 16px，隐藏滚动条并保留触控惯性滚动；底部通过 128px 内边距给输入区和渐隐遮罩预留可读空间，内容允许进入底部渐隐区域但不能被输入框遮挡。
- 项目 demo 顶部不展示「今天想问点什么～」默认问候气泡，用户消息作为当前演示对话的第一条可见消息。
- 用户气泡右对齐，宽 293px，背景 `rgba(0,0,0,0.08)`，0.5px 白色弱描边，圆角 18px，padding 12px 16px，文字 16px / 28px；气泡内有两条 0.75px 虚线，dash 2px / gap 2px，第一条位于 y=40、宽 261px，第二条位于 y=68、宽 80px。
- 回答判断态第一步参考 Figma `think 方案` node `7:535`：用户消息发送后，先在 Dots 左侧展示 `cloud.json` 判断 Lottie，而不是普通 loading。判断态外框尺寸 52×50px，背景位于 x=-10 / y=-2，尺寸 72×54px；cloud 图标 36×36px，位于外框内 x=8 / y=7；正文隐藏，不显示可见文字。
- 回答判断态第二步参考 Figma `think 方案` node `7:512`：cloud 动画、脑雾背景和简单判断文案同步开始，不需要先停顿。cloud 图标位置和第一步完全一致，仍位于外框内 x=8 / y=7，避免状态切换跳闪。脑雾外框从 52×50px 跟随文字输出进度横向展开，最终为 303×50px；背景位于 x=-10 / y=-2，最终尺寸 323×54px；正文位于 x=44 / y=13，最大宽 243px，高 24px，文案为「正在构思上海三日行程安排与景点推荐」，按字符流式输出。扫光只在文案完整输出后触发，使用低对比、慢速横向扫光。

- 正式回复内容参考 Figma `think 方案` node `112:3756`：`上海三天、轻松节奏、预算不设限，那就往“慢而精”的方向做：先把当前的顶级去处捞出来，排成一条动线。`

- 紧凑 think 过程参考 Figma `think 方案` node `113:3901`：toolcall 完成后同一条脑雾渐变切回 think，动画回到 `cloud.json`。容器最终宽 232px、高 50px；背景相对容器外扩 10px / 6px，透明度 80%；左上图标 36×36px，位于容器内 x=8 / y=7；标题位于容器内 x=44 / y=13，最大宽 188px，14px / 24px，文案为「浏览旅行目的地与预定信息」，只流式输出标题，不展示正文。
- 单行紧凑 think 没有正文，必须保持 50px 高度；不能因为通用内容流 padding 被撑到 54px 或更高。
- 第二段 toolcall 过程参考 Figma `think 方案` node `70:2394`：紧凑 think 完成后同一条脑雾渐变切换到 toolcall，动画使用 `glass.json`。容器最终宽 361px、高 159px，背景相对容器外扩 10px / 6px；标题位于容器内 x=44 / y=15，最大宽 273px，14px / 20px，文案为「搜索上海酒店真实入住体验」；正文位于容器内 x=16 / y=45，最大宽 337px，12px / 18px，gap 2px，结果区高度必须包含行间 gap，不能裁切最后一行。前两行使用 `note_single_b` 12×12 图标 +「笔记」+ 10px 高分隔线 + 内容；第 3、4 行使用 `internet_b` 12×12 图标 + 网页标题；最后一行只展示「更多内容检索中...」。标题和正文都按字符流式输出，只有标题在完整输出后扫光。
- 第三段 think 过程参考 Figma `think 方案` node `114:4085`：检索 toolcall 完成后同一条脑雾渐变切回 think，动画使用 `cloud.json`。容器最终宽 361px、高 115px，背景相对容器外扩 10px / 6px；标题位于容器内 x=44 / y=15，最大宽 273px，14px / 20px，文案为「现在为用户创建一份3天的奢华行程安排」；正文位于容器内 x=16 / y=45，最大宽 329px，12px / 18px，文案为「包括宝格丽酒店住宿、Ultraviolet晚餐和新荣记等高端体验，核心思路不是“景点打卡”，而是：老上海建筑肌理 + 当代艺术 + 顶级酒店体验 + 私人专车 + 稀缺餐厅。」标题和正文都按字符流式输出，只有标题在完整输出后扫光。
- response 过程参考 Figma `think 方案` node `7:717` 与完成态 node `157:6748`：第三段 think 完成后停顿 1s，思考脑雾先整体收束，再在同一对话流位置过渡为最终 response 卡片。response 卡片不做位移，只从 0.98 透明度柔和淡入，时长 900ms；卡片先显示顶部状态「已思考完成，参考小红书与全网内容」并停留约 420ms，再开始正文字符流式输出。卡片不能吸底，正文增长时必须沿正常文档流向下拓展，不能把已出现回答持续往上顶。卡片宽 361px，白底，0.5px `Separator` 描边，圆角 22px，padding 20px 16px；顶部状态 14px / 20px，颜色 `Description`，右侧保留叠头像和箭头。正文整体 16px / 28px，section 之间 gap 20px；section 标题 16px / 24px Semibold，标题与正文 gap 14px；「必须提前订的三样」列表项之间 gap 12px，bullet 行必须按 22px bullet 区 + 307px 文本区对齐，不允许额外 gap 挤压文本宽度。
- response 卡片必须使用设计稿里的图片资源：状态行头像叠放、右箭头、底部复制/分享/不喜欢/刷新图标都必须是真实切图，不允许用文本符号占位。底部复制/分享/不喜欢/刷新操作栏只在 response 正文完整流式输出结束后展示，不能在流式输出过程中吸附在卡片底部。
- response 顶部状态行点击后参考 Figma `think 方案` node `8:1598` 打开思考/来源半层。半层在当前 393×852 画板内从底部拉起进入，使用 `translateY(100%) -> 0` 的 420ms 柔和缓动 `cubic-bezier(0.16, 0.84, 0.32, 1)`；遮罩同步淡入。半层高 639px，背景 `Bg0 Lighter / #FAFAFA`，顶部圆角 38px，阴影 `0 15px 37.5px rgba(0,0,0,0.18)`；背景页面加 20% 黑色遮罩。顶部 grabber 位于 x=178.5 / y=5，36×5px，圆角 100px，颜色 `#CCCCCC`。半层内容从 y=20 开始，左右 padding 12px，模块间距 20px。标题区为「思考过程」，padding `4px 8px 12px`，16px / 24px Medium。思考过程卡片白底、圆角 16px、padding 12px；每行左侧进度列宽 24px，状态圆 24×24px、`rgba(48,48,52,0.05)`，连接线为 1px 竖向虚线，dash 1.5px / gap 1.5px，颜色 `rgba(20,20,20,0.08)`；文本 13px / 20px Medium，右侧 16px chevron。参考来源模块标题为「参考来源 · 128」，下方筛选胶囊和 Source Item 卡片按设计稿展示；当来源列表滚动到笔记区域时，「参考来源 · 128」标题和筛选胶囊组成的导航需要吸顶在半层滚动容器顶部，背景同半层 `#FAFAFA`，防止内容穿透。卡片白底、圆角 16px、padding `12px 12px 16px`，图片 3 列、gap 6px、圆角 6px。
- 半层里的思考过程文字色不能沿用对话流脑雾的 `Description` 弱层级；标题使用 `Title` / 13px / 20px Medium，正文、检索行、网页行和更多入口使用 `Paragraph` 层级。`搜索上海酒店真实入住体验` 展开内容中的网页来源标题严格按 Figma node `116:6095`：`PingFang SC Regular`、12px / 18px、字距 0.24px、单行展示，超出一行必须用 ellipsis 截断。
- 思考过程中，正在展示的思考脑雾整体都可点击，点击后参考 Figma `think 方案` node `8:1257` 从底部拉起「思考中」半层。半层复用同一套 639px 底部拉起、20% 遮罩、grabber、白底过程卡片和 24×24 状态列；标题为「思考中」，如果半层保持打开且主流程已经进入 response / complete，标题自动更新为「思考结束」。内容只展示当前已经走到的 think/toolcall 链路，数据必须来自前面对话里的流式过程；状态圆内的 icon 必须来自对应 Lottie JSON，think 使用 `cloud.json`、toolcall 使用 `pen.json`、search 使用 `glass.json`，非当前行停在 JSON 第一帧，当前最新一行才播放动画，尺寸按列表状态位 24×24px。半层内标题、正文和检索结果都按正常文档流向下展开，不能使用脑雾里的绝对定位或固定高度裁切内容。`搜索上海酒店真实入住体验` 行右侧不展示叠头像；思考结束后只更新半层标题为「思考结束」，该行仍保持进行中的检索列表样式，不切换为最终结果卡。
- 参考来源列表底部数据必须严格按 Figma `think 方案` node `116:4504`：第一条作者 `Yt`，标题「可是上海就是不缺有意思的地方啊」，正文两行，三张图，`来自笔记 2天前`，点赞 `2387`；第二条作者 `sailormmoon`，标题「上海宝格丽妥妥top1」，正文「上海宝格丽服务真的太好了，一分钱真一分货」，三张图，`来自笔记 2天前`，点赞 `2387`；第三条作者 `alice`，评论正文两段，无图片，`来自评论 2天前`；第四条作者 `Breeze`，评论正文两行，无图片，`来自评论 2天前`。笔记与评论正文左侧均使用设计稿的引号图标，不能用文本符号代替。
- response 状态行打开的「思考过程」半层里，`搜索上海酒店真实入住体验` 类目展开后仍使用前面对话流中的检索列表样式，不展示右侧叠头像，也不切换为结束态结果卡。
- 思考/来源半层上滑后参考 Figma `think 方案` node `12:1153` 进入高半层状态，高度从 639px 柔和过渡到 798px；不是重新打开第二个弹层。拖拽把手、半层内容向上移动，或在半层内容区产生向下浏览的滚动意图时，应在手势过程中立即切换为高半层，不等松手后才变化。半层向下拖动超过 96px 时直接关闭半层，不先退回低半层。半层打开时所有 think/toolcall 类目默认收起；单个类目支持点击展开/收起，多个类目可以同时展开，点击另一个类目不能自动收起前一个类目；点击行本身不能改变半层高度。
- 半层类目点击展开内容时，正文区域不能硬切出现；内容区使用“呼吸展开”：从 `max-height: 0` 到内容高度柔和展开，同步 `opacity 0 -> 1`，不做上下位移，时长 400ms，曲线 `cubic-bezier(0.22, 0.61, 0.36, 1)`。
- 半层过程列表的展开能力由真实内容决定：只有存在正文、检索结果或来源结果的类目才显示右侧 chevron 并支持点击展开；像 `thinkCompact` 这种只有标题、没有正文的过程节点必须作为静态节点展示，不显示箭头，也不绑定点击。
- 半层里的思考过程数据必须和前面真实流式链路对应：`think`「上海三日行程，融合奢华餐饮与住宿选项」对应第一段深度思考正文；`toolcall`「整合酒店建议、天气提示与预定指南」对应第二段工具调用正文；`thinkCompact`「浏览旅行目的地与预定信息」对应紧凑思考标题；`toolcallSearch`「搜索上海酒店真实入住体验」对应检索结果；`thinkPlan`「现在为用户创建一份3天的奢华行程安排」对应最终行程规划思考。不得为了半层单独编一套和对话流无关的数据。
- response 正文完整输出后进入 complete 态时，页面保持当前阅读位置，不自动跳到回答结尾；只有点击“至底”悬浮按钮才滚动到回答底部。
- think 和 toolcall 必须连贯在同一条脑雾里出：上一个阶段正文输出完成后停顿 1s，再切到下一个阶段。阶段切换时禁止把上一阶段长文本和下一阶段短文本叠在一起做过渡，也禁止按每个字符持续拉伸脑雾背景；这两种都会造成文本和脑雾背景长度不匹配或出现卡顿。切换瞬间应只保留一套当前文本和一套当前背景：旧阶段结束后清空标题/正文，下一阶段从空文本开始流式输出，脑雾使用当前阶段稳定宽度（常规 361px，紧凑 232px）。阶段切换不做 blur、缩放或残影叠层，只允许当前内容轻淡入；轻浮现位移控制在 1px 内，背景叠加 220ms 渐隐渐显，避免背景弹跳或大幅拉伸。该效果只作用于左侧手机里的真实 `think -> tool call -> think` 阶段切换，不改变数据顺序、进度条节点或 response 样式。
- think / toolcall 正文在流式输出时必须使用左对齐，禁止 `text-align: justify`。两端对齐会在字符追加和换行时重新分配字间距，导致内容间距抖动。
- 所有脑雾背景展开必须是缓慢连续的缓动，不允许随字符或行数一格一格跳动；横向宽度变化使用柔和曲线 `cubic-bezier(0.16, 0.84, 0.32, 1)`，纵向高度由正常文档流内容自然撑开，不用固定高度兜底。
- 判断动画结束后，同一位置替换为 Dots 回复气泡，文本按字符连续打出，形成完整 typewriter 动作；不显示硬光标、尾点或闪烁描边。
- 底部输入区固定在页面底部，输入区背后使用独立透明渐变遮罩层：渐变层允许向左右和底部外扩，并由最外层手机容器统一裁切，避免内层背景露出额外圆角；渐变使用 `#F5F5F5` 三段 stop：`0% / 0%`、`18% / 80%`、`36% / 100%`，36% 后保持 100% 到底部；底部渐变底座不使用背景模糊；输入框、AI 标注和 home indicator 必须位于该遮罩层之上。
- 输入框高度 48px，左右 margin 12px，padding 12px 14px，圆角 16px，背景纯色 `#FCFCFC`，阴影 `0 0 36px rgba(0,0,0,0.08)`，输入框自身保留 25px backdrop blur 的合成层。输入框父级不能使用 `isolation: isolate` 这类隔离合成上下文，否则生产浏览器可能只采样到底部渐变层，导致线上看不到对话内容的背景模糊。
- 占位文案 16px / 24px，颜色 `Description`；右侧相机和添加入口间距 16px。

- `conversation-streaming` 项目 demo 当前用于动作验证，页面右侧只保留流程进度控制器：标题「回答loading新增长思考模式」放在进度条上方，使用首页展示字体栈 `var(--font-ui)`，22px / 30px / Semibold，居中。标题下方不展示流式输出切换控件，流式输出仅保留 B 版；B 版参考 Claude 输出观感做尾部透明度流式：应用范围包括判断文案、context 回答气泡、think/toolcall 脑雾和最终 response。B 版规则是：文本仍按真实字符流进入文档流，不使用 Canvas 或白色遮罩；正在输出时只有最后约 10 个字符进入透明度梯度，最后一个字符约 34% 透明度，向前逐步过渡到 100%，每次新字符进入时旧尾部字符通过 260ms ease-out 逐步变实；输出完成时不能直接跳到 100%，而是保留尾部梯度并用 760ms 柔和动画慢慢回到 100% 透明度。think/toolcall 中标题和正文分阶段处理：标题尾部透明度完成后，正文或检索结果再用 180ms 延迟淡入进入，避免标题和内容同时抢焦点。该方案用于减少逐字跳出感，同时避免遮罩覆盖整段内容造成文本和背景错位。控制器不使用卡片背景、描边或阴影，也不显示顶部编号/当前状态标题行；进度条包含 10 个可点击节点，顺序为「判断 / think / context / think / tool call / think / tool call / think / response / 完成」，其中 context 是第一条回答气泡；已完成进度必须准确推进到当前节点圆心。点击节点直接跳到对应 demo 状态，并从该节点继续自动播放后续内容，不停在单个节点。初始状态只显示小尺寸主按钮「开始演示」；点击后从头不间断自动播放，并切换为两个小按钮：「暂停」为主按钮，「重新开始」为 36×36px 次级图标按钮，图标使用 ReDs Icons / `refresh_b`。点击暂停后必须冻结当前帧，不重置到节点开头，主按钮文案改为「继续」；点击继续后从当前流式位置恢复播放。点击重新开始后从头继续自动播放。进度条和按钮都在控制区域内水平居中；当页面进入上下布局时，手机页面、进度条和按钮必须共用同一条居中轴线，整体在可用视口内垂直居中，不保留额外底部偏移。右侧大段说明、标签、旧的「分步骤 / 一口气出完」按钮和下方状态说明模块先不展示。
