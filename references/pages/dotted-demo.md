# 点点demo

## 需求背景

点点demo 是 Dots 独立设计系统的第一个页面样本，用来验证「Figma 规范 → 规格文档 → 页面 demo → GitHub → Vercel 预览」这条流程能否跑通。

视觉以 Figma `Dots 规范` 为准：默认历史消息态参考节点 `1421:1089`，点击键盘后的输入态参考节点 `1419:970`，选中 Skill 后的输入态参考节点 `1422:2458`。代码不得脱离 Figma 和本文档自行改版。

## 页面结构

1. 状态栏
2. 顶部导航：左侧返回，居中标题「点点」，右侧更多
3. 空白对话区
4. 底部横向快捷提示：穿搭指南、好物买手、旅行攻略、上帝摄像机
5. 底部输入栏：添加、按住说话、键盘入口、相机入口
6. AI 生成标注
7. 输入态：Dots 历史消息流、输入占位、工具按钮、系统键盘
8. 选中 Skill 输入态：Skill 名称、关闭入口、输入框、工具按钮、系统键盘

## 视觉规格

- 画板宽度 393px，高度 852px。
- 页面背景为浅灰底，顶部和内容区不新增卡片或分割线。
- 中部保持大面积空白，不新增内容卡片。
- 底部输入状态整体高度 144px。
- 快捷提示行高度 40px，左右边距 20px，item 间距 8px。
- 快捷提示为白色 pill，高 40px，左图标 20px，文字 14px Regular，字距 0.56px。
- 输入区高度 96px，背景为浅灰，顶部圆角 28px，顶部内边距 10px。
- 语音输入行高度 48px：左侧添加区域 54px，中间语音按钮自适应，右侧键盘/相机区域 76px。
- 语音按钮为白色圆角矩形，圆角 16px，文字「按住说话」居中。
- 底部水印高度 38px，显示「内容由 AI 生成」和 iOS home indicator。
- 点击键盘入口后进入输入态：输入面板从 y=393.274px 开始，高 458.726px。
- 历史消息流按 Figma `1421:1089` 展示 Dots 消息体：User 右侧灰底气泡、Dots 左侧白底气泡、AI 生成卡片。
- 历史对话文案围绕 AI 生成卡片内容模拟真实护肤咨询：用户询问黑绷带面霜用法、厚涂和频率，Dots 先给简短结论，再展示生成卡片。
- 消息内容区为独立纵向滚动容器：默认态高度 575px，输入态高度 264px，选中 Skill 输入态高度 240px；隐藏滚动条，保留触控惯性滚动。
- 键盘拉起、选中 Skill、发送新消息后，消息内容区自动滚动到底部，保证最新消息露出在输入面板上方。
- 输入态「给点点发消息」区域为真实输入框，可输入文本。
- 输入态底部键盘使用 Figma 导出的本地键盘整图。
- 输入态点击上方空白对话区关闭键盘，回到默认底部栏，保留已输入内容。
- 输入态有文本内容时，右侧语音菜单和相机入口替换为发送按钮。
- 点击底部快捷提示胶囊后自动拉起输入态，并在输入框上方展示已选 Skill。
- 已选 Skill 行：左侧 20x20 图标 + Skill 名称，右侧 24x24 关闭入口；点击关闭入口只移除已选 Skill，不关闭输入态。
- 点击发送后追加到历史消息流并自动收起键盘：用户消息在右侧，Dots 回复在左侧，样式参考 Figma `1421:1089`；若本次发送带已选 Skill，发送完成后清空已选 Skill。
- 用户消息气泡：右对齐，最大宽 293px，背景 `#E6E6E6`，0.5px 白色 75% 边框，圆角 22px，内边距 12px 16px。
- Dots 回复气泡：左对齐，最大宽 293px，背景白色，0.5px 白色 20% 边框，圆角 22px，内边距 12px 16px。
- 消息文字：PingFang SC Regular，16px，行高 26px，字距 2%。
- AI 生成卡片参考 Figma `1421:1105`：宽 353px，白色 80% 背景，圆角 36px，内边距 26px 22px 32px，内容间距 26px；顶部三张 105px 圆角图片，下方为「正确使用手法」和「注意事项」内容块。
