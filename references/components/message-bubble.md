# MessageBubble 消息体

> status: complete
> last_updated: 2026-07-08
> 真相源：Figma `think 方案` node `3:4417`、`references/pages/dotted-demo.md`。

消息体是点点对话页的基础单元，承载用户消息、Dots 文本回复、判断态、流式输出和后续富文本卡片。页面 demo 只能引用该组件规范，不允许复制一份私有气泡样式。

---

## Harness 定义

MessageBubble 是点点对话流的核心 Component Harness，不是普通气泡样式。所有用户消息、Dots 文本回复、判断态和流式输出都必须通过消息体进入对话流。

- 语义：表达一次对话中的消息单位，负责“谁说的、当前处于什么阶段、展示什么内容”。
- 生成规则：AI 生成对话页时必须先选 `role` 和 `state`，再选择普通文字、判断态或流式回复，不能手写私有 bubble。
- 验证方式：检查左右对齐、背景、圆角、连续消息间距、发送后滚动到底、判断态到流式态的替换。

---

## 组件结构

```tsx
<DotsMessage role="user | dots" state="default | thinking | streaming | complete">
  <DotsMessageBubble />
</DotsMessage>
```

`DotsMessage` 负责左右对齐、消息间距和生命周期位置；`DotsMessageBubble` 负责文字气泡、判断态容器和逐字流式文本。

AI 富文本卡片仍属于消息流内容，但当前 `think 方案` 页面基座不展示 AI 卡片。需要卡片时必须先补对应组件规格，再放回对话流。

## 判断态

AI 回答过程的第一阶段是 Dots 左侧判断态，不使用骨架屏、硬 loading 圆点或品牌色描边。

| 属性 | 值 |
|------|------|
| 文案 | 第一步正文隐藏；第二步显示简单判断文案 |
| 尺寸 | 第一步外框 52×50px、背景 72×54px；第二步外框从 52×50px 跟随文字输出展开到 303×50px，背景最终 323×54px |
| 背景 | 白色模糊底，blur 8px |
| 圆角 | 22px |
| 动画 | 当前只播放一个 36×36 `cloud.json` 判断 Lottie；第一步和第二步的 Lottie 都位于外框内 x=8 / y=7，不发生位移跳闪 |
| 文字 | 第二步文案为「正在构思上海三日行程安排与景点推荐」，位于 x=44 / y=13，最大宽 243px，高 24px，并按字符流式输出；扫光只在完整输出后触发，停留期间带低对比、慢速横向扫光动画 |
| think | 正式回复完成后追加 361×121px 思考脑雾；背景相对容器左右各外扩 10px、上下各外扩 6px；标题位于容器内 x=44 / y=13，14px / 24px；正文位于容器内 x=16 / y=47，12px / 18px，页面左右边距均为 32px，必须允许正常换行；标题和正文都流式输出，只有标题在完整输出后扫光 |
| toolcall | think 完成后同一条脑雾渐变切换到 toolcall，动画使用 `pen.json`；容器最终 361×155px；标题位于容器内 x=44 / y=15，14px / 20px；正文位于容器内 x=16 / y=45，12px / 18px；标题和正文都流式输出，只有标题扫光 |
| 紧凑 think | toolcall 完成后同一条脑雾渐变切回 think，动画使用 `cloud.json`；容器最终 232×50px，背景透明度 90%；标题位于容器内 x=44 / y=13，14px / 24px；只流式输出标题，不展示正文 |
| toolcall 检索 | 紧凑 think 完成后同一条脑雾渐变切换到 toolcall，动画使用 `glass.json`；容器最终 361×159px；标题位于容器内 x=44 / y=15，14px / 20px；正文位于容器内 x=16 / y=45，12px / 18px，逐行展示检索结果；标题和正文都流式输出，只有标题扫光 |
| 行程 think | 检索 toolcall 完成后同一条脑雾渐变切回 think，动画使用 `cloud.json`；容器最终 361×115px；标题位于容器内 x=44 / y=15，14px / 20px；正文位于容器内 x=16 / y=45，12px / 18px；标题和正文都流式输出，只有标题扫光 |
| response | 行程 think 完成后停顿 1s，思考脑雾先消失，再追加最终 response 卡片；卡片 361px 宽，白底，0.5px 描边，22px 圆角；顶部展示「参考小红书与全网内容」状态行，正文 16px / 28px 按字符流式输出；卡片像普通气泡一样从下方 32px 柔和浮现，并在正常文档流内向下拓展；底部操作栏只在正文完整输出结束后展示 |

判断态必须占用 Dots 回复位置。首段回答到达后，判断态替换为同一条 Dots 回复气泡，不新增第二条 loading 消息。

## 流式输出

AI 回答生命周期为 `thinking -> judging -> streaming -> think -> toolcall -> think -> toolcall -> think -> response -> complete`，其中 `thinking` 在当前页面表现为 Lottie 判断过程。

| 阶段 | 行为 |
|------|------|
| thinking | Dots 左侧展示 cloud 判断 Lottie |
| judging | cloud 位置保持不变，脑雾背景跟随右侧简单判断文案逐字输出同步展开 |
| streaming | 同一条 Dots 回复气泡内按字符连续追加文本 |
| think | 正式回复完成后追加带标题和正文的思考脑雾，标题和正文都按字符流式输出，只有标题扫光 |
| toolcall | think 完成后渐变切换到 toolcall，动画切换为 pen，标题和正文都按字符流式输出 |
| think | toolcall 完成后渐变切回紧凑 think，动画切回 cloud，只流式输出标题 |
| toolcall | 紧凑 think 完成后渐变切换到第二段 toolcall，动画切换为 glass，标题和检索结果都按字符流式输出 |
| think | 检索 toolcall 完成后渐变切回行程 think，动画切回 cloud，标题和正文都按字符流式输出 |
| response | 行程 think 完成后停顿 1s，隐藏思考脑雾并追加最终 response 卡片，正文流式输出；流式过程中不展示底部操作栏 |
| complete | 保留普通 Dots 回复气泡和完整 response 卡片；response 操作栏在正文输出结束后展示，但不自动跳到回答结尾 |

简单判断和正式回复必须连接为一个连续动作：`judging` 文案完成后进入 `judgingHold` 保持 2s，再直接进入 `streaming`。正式回复气泡从下方 32px 位移浮现，透明度从 0.94 到 1，使用专用柔和曲线 `cubic-bezier(0.16, 0.84, 0.32, 1)`，时长 1200ms；文字流式输出与气泡浮现同步开始。think 和 toolcall 阶段之间必须在同一条脑雾里连贯切换：上一个阶段正文输出完成后停顿 1s，再进入下一个阶段，只保留同一脑雾容器自身的渐变。进入 response 时，思考脑雾消失，response 卡片使用同一套 32px / 1200ms 浮现动效，并随正文向下拓展；不要在每个字符输出时强制把内容顶到视口底部。思考脑雾背景透明度为 90%，横向宽度展开使用缓慢连续缓动，纵向高度必须由正常文档流内容自然撑开，不允许用字符数/行数估算固定高度。流式输出参考 typewriter 实现方式：用固定 `typeSpeed` 按字符推进文本，不展示硬光标、尾点或闪烁描边；完成后视觉上就是一条普通 Dots 回复。

## 消息流间距

| 场景 | 值 |
|------|------|
| 单条消息行上下 padding | 4px 0 16px |
| 对话流左右 padding | 16px |
| 对话流顶部 padding | 12px |
| 时间戳 | 居中，12px / 17px，`Placeholder` |

时间戳作为消息流里的独立 stream item，不属于 User 或 Dots 气泡。会话首条消息前显示完整时间锚点；同一连续会话内不重复展示；当相邻消息间隔达到 15 分钟或跨天时，在下一条消息前插入新的时间戳。

## 交互

| 交互 | 行为 |
|------|------|
| 点击气泡 | 不改变气泡样式 |
| 长按气泡 | 打开长按面板，面板样式另行定义 |
| 发送新消息 | 新消息追加到底部，消息流滚动到最新消息 |
| AI 回答判断中 | 展示 Dots 左侧判断过程 Lottie |
| AI 回答流式输出 | 同一条 Dots 气泡持续追加文本，滚动到最新内容 |
| response 超出可视区域 | 在输入区上方显示“至底”悬浮按钮，点击平滑滚动到回答底部并隐藏；按钮本体和箭头不做额外动画 |
| 点击思考脑雾 | 打开 Figma `8:1257`「思考中」半层；半层展示当前已经走到的 think/toolcall 链路，状态位统一使用对应 Lottie JSON，非当前行停在第一帧，最新一行播放动画；内容按正常文档流向下展开；若半层打开期间思考结束，标题更新为「思考结束」，`搜索上海酒店真实入住体验` 切换为 Figma `116:6060` 的结果卡样式 |
| 点击 response 状态行 | 打开 Figma `8:1598` 思考/来源半层，展示思考过程和参考来源列表 |
| 思考/来源半层内容上滑 | 在手势过程中进入 Figma `12:1153` 高半层状态，高度柔和展开；所有类目默认保持收起 |
| 思考/来源半层下拉 | 向下拖动超过 96px 时直接关闭半层 |
| 点击思考类目 | 展开或收起当前类目的过程说明；多个类目可以同时展开，点击另一个类目不自动收起前一个；展开内容使用 400ms 高度/透明度“呼吸展开”，不做上下位移，不硬切 |
| 无内容思考类目 | 不显示展开箭头，也不绑定点击；仅作为过程时间线里的静态节点 |

按压反馈统一使用轻量 scale 与透明度变化；icon、图片禁止拖动。
