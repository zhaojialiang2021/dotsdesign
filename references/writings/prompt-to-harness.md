# 从 Prompt 到 Harness

> AI Agent 时代的人机协作，正在从表达意图走向组织执行。

过去一段时间，我们谈 AI 使用能力，最常提到的是 **Prompt Engineering**。它的核心是：如何把需求说清楚，让 AI 更好地理解我的意图。

这当然重要。Prompt 并没有失效，也不会被取代。它依然是人与 AI 协作的入口，只是更擅长解决“表达意图”的问题。

但当任务走向持续执行，一个更值得关注的概念出现了：**Harness Engineering**。它代表的不是对 Prompt 的否定，而是协作方式的升级：从表达意图，走向组织执行。

## Prompt 解决“我想要什么”，Harness 解决“如何稳定完成”

如果任务足够简单，写清楚一句话就够了。但当你希望 AI 持续产出符合品牌调性的插画，或让 Codex 修复真实项目里的 bug，它需要的就不止是理解一句话。

Prompt 的核心问题是：

> 我怎么把需求告诉 AI？

比如：帮我生成一张社区公约插画，横图、蓝色调、抽象一点、不要文字。它帮我们清楚地表达目标、风格、限制和输出格式。

Harness 的核心问题是：

> 如何让它稳定地完成？

它需要知道上下文在哪里，知道能调用哪些工具，知道哪些规则不能破坏，知道怎么验证结果，知道失败后如何继续修正，也知道人类会以什么标准审查。

**Prompt Engineering 是把一次对话问好。**

**Harness Engineering 是把一套工作流搭好。**

两者不是替代关系，而是场景分工：Prompt 更适合表达意图，Harness 更适合组织执行。

## 为什么 Agent 时代更需要 Harness

因为真实工作不是一次回答能完成的。以 Codex 这类编程 Agent 为例，它要做的往往不是“写一段代码”，而是理解结构、定位文件、修改、跑测试、看报错、继续修复、生成 PR、等待 review。

这背后是一个循环：

```
Plan → Act → Observe → Reflect → Repeat
```

普通 AI 是“你问，我答”。Agent 更像“你给目标，我拆解、执行、观察、修正，直到结果可验证”。

而这一切要成立，前提是它拥有代码仓库、测试命令、终端日志、权限边界与项目文档。否则它只能“想象怎么做”。

Harness 的价值，就是把 Agent 放进一个能工作的环境里，让这个循环跑得起来，而且跑得稳定。

## 不是人被替代，而是人的位置变了

在 Agent 工作流里，人越来越像是定义目标的人、设定标准的人、设计流程的人、判断好坏的人、做最终取舍的人。

Agent 则负责查资料、生成方案、调用工具、跑流程、反复试错、执行大量具体动作。

所以人的价值不是消失，而是从“做每一步”，迁移到“定义什么值得做，以及做到什么程度才算好”。

> Humans steer. Agents execute.

人类掌舵，Agent 执行。这不是人退场，而是人从“亲自做每一步”，迁移到“定义什么值得做，以及做到什么程度才算好”。

## 对设计师来说，Harness 是一种“设计生产系统”

设计师一直在做的事，本质上就是为复杂关系建立秩序。不是只设计按钮，而是设计行为路径；不是只设计页面，而是设计用户和系统之间的关系。

AI 时代也一样，工作会多出一层：不只是做结果，而是搭建结果产生的机制。

比如你希望 AI 持续生成符合品牌感的插画，光写 prompt 远远不够，你真正需要的是一套 harness：

- 哪些颜色可以用，哪些构图不能出现
- 人物比例怎么控制，是否允许出现文字
- 什么叫“高级”，什么叫“太复杂”
- 哪些元素是不可动的品牌资产
- 输出后怎么筛选
- 怎么把反馈沉淀成下一轮规则

这就不是简单的“提需求”，而是在设计一个 AI 时代的创作系统。

设计师的能力，会从一个问题升级到另一个问题：

> 我能不能做出一个好方案？
>
> 我能不能搭出一套持续产生好方案的系统？

## Prompt 仍然重要，但它只是协作系统的一部分

这里要避免一个误解：Harness 并不是说 Prompt 不重要了。Prompt 依然是人与 AI 协作最直接的入口。

只是当任务从“生成一个结果”走向“持续完成一个复杂目标”，影响结果的因素会变多：

- 上下文是否完整
- 工具是否可用
- 规范是否清楚
- 反馈是否及时
- 结果是否可验证
- 人能否有效审查
- 系统能否把经验沉淀下来

**Prompt 是入口，Harness 是系统。**

**Prompt 帮 AI 理解你，Harness 帮 AI 和你一起完成事。**

所以未来真正拉开差距的，可能不是谁会写一句更漂亮的 prompt，而是谁能把 AI 放进一个更好的工作系统里。

## 我的核心判断

如果 Prompt Engineering 是 AI 早期使用者的关键技能，那么 Harness Engineering 可能是 AI Agent 时代更重要的组织能力。

它不是替代 Prompt，而是把 Prompt 放进一个更完整的协作系统里。

因为 AI 越会执行，人的价值越不在于亲手完成所有动作，而在于：**设定方向、组织资源、建立标准、设计反馈，并判断什么才是好结果。**

未来的设计师，可能不只是界面的生产者，也不只是 prompt 的使用者，而是 **AI 工作系统的设计者**。

真正重要的不是“怎么命令 AI”，而是怎么设计一个让 AI、人、工具和反馈一起工作的系统。

## 结语

**Prompt 让 AI 听懂你。**

**Harness 让 AI 真正和你一起工作。**

AI Agent 时代，设计师真正要关注的，不是 Prompt 是否过时，而是：在不同场景里，如何把表达意图、组织执行、反馈验证和人的判断，连接成一个稳定的协作系统。
