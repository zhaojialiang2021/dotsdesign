> status: draft

## 目标

入门页合并原「介绍」和「宣言」，作为 System 区的第一入口。它不再拆成两个页面：介绍负责讲清楚 Dots 是什么，宣言负责讲清楚为什么 AI-native 设计系统需要 Harness Engineering。

## 叙事顺序

1. Hero：给 AI 也设计的设计系统。
2. 为什么要重做设计系统：文档读了也用不上、AI 放大偏差、约定升级成契约。
3. Dots 的答案：Token / Component / Pattern / Page / Agent Harness。
4. 三类读者：产品设计师、开发、AI agent。
5. 验收标准：token 使用率、props 命中率、状态完整性。
6. 从哪里开始：工作流、令牌、原则、组件、页面、触觉反馈、项目汇报。

## 导航规则

- 侧栏「入门」只保留「介绍」和「设计工作流」。
- `/docs/manifesto` 作为旧链接兼容，进入合并后的 `/docs/intro` 内容。
- Writing 区可保留长文版 manifesto，不作为入门导航入口。
