# Dots 三 Tab 架构

底部三 tab 导航定义了 Dots 的所有核心交互路径。三个 tab 不是功能的分类，是用户使用阶段的三个节点：记 → 停 → 看。

---

## 总览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Tab 1     │    │   Tab 2     │    │   Tab 3     │
│   记录       │ ──→│   洞察       │ ──→│   我         │
│   河流       │    │   摆渡       │    │   港湾       │
│             │←───│             │    │             │
│   记        │    │   停、看     │    │   找、管     │
└─────────────┘    └─────────────┘    └─────────────┘
  输入驱动            发现驱动            管理驱动
  零结构              聚类+联想            时间线+设置
```

三个 tab 之间不是平行关系，是一个用户旅程的循环：Tab 1 记 → Tab 2 停下来看发现了什么 → Tab 3 翻翻过去、调调设置 → 回到 Tab 1 继续记。

---

## Tab 1：记录

### 一句话

像聊天一样轻松记录。不需要分类，不需要标签，不需要结构。

### 用户来做什么

- 随手记下一闪而过的想法
- 丢一段语音、一张图、一个链接
- 看到 AI 的回应，追问下去
- 跟 AI 聊天式地梳理一个想法

### 页面栈

```
记录 (首页)
├── 消息流 (用户气泡 + AI 回应气泡混排，按时间)
├── 日期标记 (自动插入)
├── 输入栏 (文字 / 语音 / 附件)
└── 附件面板 (摄像头、照片、链接)
```

### 从 Tab 2 追问回到这里

用户在 Tab 2 看到一条洞察，点「追问」→ 带着那条洞察的上下文跳回 Tab 1，AI 以洞察为背景开始新的对话。这是两个 tab 之间最重要的交互。

### 设计原则

- 零分类：用户不需要决定「这个想法属于哪个文件夹」
- 零评判：AI 不评价用户记录的质量
- 对话感：AI 气泡使用 `Fill A`，用户气泡使用 `Fill B`，用户在聊、不是在填表
- 轻触觉：发送 light，收到 AI 回应 tick

### 什么不放这里

- 不给用户看「你有 42 条未整理」——数字是压迫
- 不推荐「你还可以记这个」——系统没有资格建议用户记什么
- 不让 AI 主动发起对话——只有用户开口，AI 才回应

### 对应现有页面

`home`

## Tab 2：洞察

### 一句话

对已记录的内容进行梳理和洞察发现。系统从碎片里找模式，用户来判断哪些值得追。

### 用户来做什么

- 看看最近自己的记忆里有没有什么规律
- 发现一条自己没意识到的跨主题关联
- 追问一个洞察，跳回 Tab 1 深聊
- 忽略没价值的，标记有价值的

### 页面栈

```
洞察
├── 洞察卡片流 (AI 生成的今日洞察 × 0–3)
│   └── 每张卡片：标题 + 一句话结论 + 来源引用 + 追问/忽略/分享
├── 主题集群 (你在反复想这些事，× 3–6)
│   └── 每集群：集群名 + 记忆数 + 时间跨度，点击展开
├── 碎片河流 (未被聚类的单独记忆)
├── 洞察日历 (过去 7 天，标记有洞察的日期)
```

### 核心机制：三个 Pipeline

每天凌晨 5 点跑一次。

```
用户记忆碎片 (n 条)
      │
      ▼
Pipeline 1: 压缩
  语义聚类 + 关键词提取
  同类 ≥3 条 → 成集群
  不足 3 条 → 入碎片河流
  最大 8 个集群
      │
      ▼
Pipeline 2: 联想
  跨集群检测关联
  Shannon 门槛过滤：
    用户已知道？→ 不过
    无张力同义反复？→ 不过
    连接太弱（高频停用词）？→ 不过
  每天最多 3 条联想
      │
      ▼
Pipeline 3: 表达
  标题 ≤12 字
  结论 ≤40 字
  引用 2–4 来源
  陈述语气，不指导行动
      │
      ▼
洞察卡片呈现给用户
```

### 设计原则

- 少即是多：每天最多 3 条洞察。0 条也是正常的——承认有些天没有足够信号是诚实的，比编一个更尊重用户。
- 系统只发现，用户才判断：洞察卡片是「提问」不是「结论」。用户必须做选择（追问/忽略），系统不会替用户「自动采纳」。
- 不是仪表板：不做统计报表、不做使用时长排名、不做「你比上周多记了 30%」。这些数字把用户变成被观测对象。
- 触觉：进入 Tab 2 tick，展开卡片 light，忽略 tick。

### 什么不放这里

- 不推「推荐阅读」——Tab 2 只看已经记录的，不引入新料。哈勃半径（用户可能接触但未记录的内容）是未来的事，MVP 不做。
- 不显示统计数据——不使用时长、条数排名、连续天数。
- 不让 AI 评价用户——「你最近的记录质量有所下降」这类话永远不说。

### 对应新页面

`insights`（新建，不在现有 ScreenId 列表中）

## Tab 3：我

### 一句话

关于我的一切：我在哪、我记了什么、我怎么调的。时间线铺开，设置收起。

### 用户来做什么

- 翻自己记录过的所有东西，按时间线
- 搜索一条特定记忆
- 看 AI 生成的月度/周度总结
- 查看自己的能力资产（记忆点、可用次数等）
- 调整通知、AI 偏好、语言、深色模式
- 看帮助中心、使用条款、隐私政策

### 页面栈

```
我
├── 用户卡片 (头像 + 昵称 + ID)
├── 数据区
│   ├── 记录总数 (记忆点)
│   ├── 时间线浏览入口 → 记忆回顾页
│   ├── 月度总结入口 → 智能总结页
│   └── 能力资产入口 → 时间碎片详情页
├── 设置区
│   ├── 通知 (Switch 开关)
│   ├── AI 偏好 → AI 偏好页
│   ├── 应用语言 → "中文"
│   └── 深色模式 → "跟随系统"
├── 关于区
│   ├── 帮助中心
│   ├── 使用条款
│   └── 隐私政策
└── 底部版本号
```

### 与原有侧边栏的关系

原侧边栏（sidebar）的设置功能基本迁移到 Tab 3 的设置区。Tab 3 的「我」页面就是原侧边栏内容 + 数据入口的合并。

原有页面中：
- `sidebar` → 设置部分迁移到 Tab 3 主页面的设置区，不再需要单独滑出
- `memory-review` → 从 Tab 3 主页面的「时间线浏览」入口进入，保持独立页面
- `ai-summary` → 从 Tab 3 主页面的「月度总结」入口进入，保持独立页面
- `time-fragment` → 从 Tab 3 主页面的「能力资产」入口进入，保持独立页面
- `ai-preferences` → 从 Tab 3 设置区「AI 偏好」入口进入，保持独立页面

### 设计原则

- 敬畏时间线：所有记录默认按时间排列。不应以「相关度」重新排序，时间是最诚实的索引。
- 设置要稳：设置项不能弹出惊喜——你进来改了就走，不应该被打扰。
- 不造焦虑：不显示「你 3 天没记了」，不显示「记忆留存率下降」。
- 触觉：进入 Tab 3 tick，点击列表项 light，Switch 切换 light。

### 什么不放这里

- 不放功能推荐、不放运营 banner
- 不放社交功能（好友、排行榜）——MVP 没有社交
- 不显示 app 使用时长——这不是健康管理软件

### 对应现有页面

`me`（新建，合并 sidebar、time-fragment、memory-review、ai-summary 的入口功能）

## 三 Tab 关系：闭合环路

```
     记 (Tab 1)
        │
        │ 碎片积累
        ▼
     停 (Tab 2)
        │
        │ 追问 ←── 最重要的一条线
        ▼
     记 (Tab 1) 继续对话
        │
        │ ...
        ▼
     看 (Tab 3)
        │
        │ 翻翻过去，找找感觉
        ▼
     记 (Tab 1) 又有新想法了
```

环的驱动力：Tab 1 产生材料，Tab 2 从材料中提问题，用户追着问题回到 Tab 1，Tab 1 的新对话产生新材料。Tab 3 是这个环的休息站——不参与闭环，但用户随时可以进来看看自己走了多远。

## TabBar 视觉规格

| 属性 | 值 |
|------|-----|
| 高度 | 84px (含安全区) |
| 背景 | `Bg 1`，顶部 0.5px `Separator 2` |
| 图标 | 24px，来自 Bridge 图标库 |
| 文字 | 10px, Label-Secondary |
| 选中态 | 文字变 `Title`，图标变填充态 |
| 切换触觉 | tick |

三个 tab 图标：
| Tab | 图标 | 文字 |
|-----|------|------|
| 1 | chat / bubble | 记录 |
| 2 | spark / lightbulb | 洞察 |
| 3 | person / user | 我 |

## 断点适配

| 断点 | TabBar | Tab 3 布局 |
|------|--------|-----------|
| Mobile (390px) | 底部固定，三等分 | 单列全屏 |
| Tablet (768px) | 底部固定 | 设置区最大 540px 居中 |
| Desktop (1440px) | 可考虑侧栏导航 | 内容最大 680px 居中 |
