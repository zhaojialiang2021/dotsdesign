# Card

点点卡片分两类：对话流中的 AI 富文本卡片，以及内容块类卡片（商品、地点、社区声音、外链等）。卡片必须先判断自己是否属于消息体。

## AI 富文本卡片

| 属性 | 值 |
|------|------|
| 角色 | AI 消息体的一种，左对齐 |
| 背景 | `Fill 5` |
| 边框 | 0.5px `Separator 2` |
| 圆角 | `radius.ai-card` = 36px |
| 阴影 | `shadow.1` |
| 标题文字 | `Title` + `rich-h*` |
| 正文文字 | `Paragraph` + `rich-paragraph` |
| 标注文字 | `Description` |
| 文字左右间距 | 24pt |
| 内容块左右间距 | 14pt |

卡片内部遵循富文本内容规则：内容块不能连续出现，列表不嵌套，高亮块不能连续。

## 内容卡片

| 属性 | 值 |
|------|------|
| 结构 | 左文右图 |
| 背景 | `rgba(255,255,255,0.72)` + blur 40px saturate 180% |
| 圆角 | 22px |
| 标题 | 13px / 600，单行截断 |
| 缩略图 | 44×44px，12px 圆角；个人页用圆形 |
| 来源 | 16×16 icon + 11px 平台名 |

## 评论分享卡

| 元素 | 规范 |
|------|------|
| 外框 | `Fill 5`，圆角 22px |
| 头部标签 | `Description` |
| 正文标题 | `Title` |
| 分隔线 | `Separator 2`，5% 视觉强度 |
| 来源区 | 缩略图 + 来源名 + 标题 |

## Tool Call 来源胶囊

来源胶囊用于对话流内的轻量 tool call 反馈，不承载完整详情。

| 属性 | 值 |
|------|------|
| 背景 | 弱灰填充，不能额外加卡片底色 |
| 圆角 | pill |
| 内容 | 平台 icon / 叠图 + 数字 + 来源类型 |
| 数字 | 从 0 滚动到目标值，节奏时快时慢 |
| 出现 | 一个一个出现，不同时闪现 |

当前长思考来源胶囊数据：

- `14 篇小红书笔记`
- `8 条全网信息`

## Tool Call 信息卡

信息卡用于更具象地展示来源统计，当前用于回答 loading 的 tool call 样式切换。

| 属性 | 值 |
|------|------|
| 数量 | 两张：小红书、全网 |
| 布局 | 文本左对齐，图片右对齐，中间间距 auto |
| 对齐 | 图片和文字模块在卡片高度内垂直居中 |
| 背景 | 不额外加色块；只保留设计稿边框和必要弱阴影 |
| 动效开关 | 只控制右侧图标是否动画；默认展示但不动 |

右侧图标动画使用横向顶替：前一个图标渐隐消失，后一个图标顶到前一个位置，新的后置图标在原后置位置渐显。动画过程中必须保持两个图标的叠放关系，不能出现只剩一个图标的空档。

## 来源详情卡

来源详情卡用于半层里的完整来源列表。

| 属性 | 值 |
|------|------|
| 外框 | `Fill A`，12px 圆角，0.4px `Separator 2` |
| padding | leading 4px，trailing 8px，vertical 4px |
| 缩略图 | 44×44px，8px 圆角 |
| 标题 | 单行截断 |
| 作者 | 头像 + 名称 |
| 点赞 | 右侧心形 icon + 数字 |

## 禁止

- AI 富文本卡片不能包在普通白色气泡里。
- 内容块不能在列表、引用块、高亮块内嵌套。
- 不用普通白卡片规则替代 `Fill 5`。
- 来源胶囊不能一次性全部闪现。
- Tool call 信息卡不能用大面积背景色替代设计稿里的轻描边卡片。
